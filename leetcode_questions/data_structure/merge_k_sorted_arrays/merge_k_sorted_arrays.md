# 合并K个有序数组

## 题目描述
给定K个有序数组，将它们合并成一个有序数组。

## 示例
输入:
```python
[
  [1, 4, 7],
  [2, 5, 8],
  [3, 6, 9]
]
```
输出: [1, 2, 3, 4, 5, 6, 7, 8, 9]

## 解题思路

### 方法一：最小堆
1. **核心思想**
   - 使用最小堆维护K个数组当前的最小值
   - 每次取出堆顶元素（全局最小值）
   - 将该元素所在数组的下一个元素入堆

2. **实现细节**
   - 堆中存储元组(值, 数组索引, 元素索引)
   - 通过数组索引和元素索引定位下一个元素
   - 使用Python的heapq模块实现最小堆

3. **复杂度分析**
   - 时间复杂度：O(NlogK)
     - N是所有元素总数
     - K是数组个数
     - 每个元素入堆出堆一次
   - 空间复杂度：O(K)
     - 堆的大小为K

### 方法二：分治法
1. **核心思想**
   - 将K个数组分成两半递归处理
   - 最后合并两个有序数组
   - 类似归并排序的思路

2. **实现细节**
   - 递归合并数组范围
   - 基本情况是合并两个数组
   - 使用双指针法合并有序数组

3. **复杂度分析**
   - 时间复杂度：O(NlogK)
     - N是所有元素总数
     - K是数组个数
     - 递归树的高度是logK
   - 空间复杂度：O(N)
     - 需要存储中间结果

### 两种方法的比较
1. **堆方法**
   - 优点：空间效率更好
   - 缺点：需要额外的数据结构

2. **分治方法**
   - 优点：实现直观，易于理解
   - 缺点：需要额外的空间存储中间结果

## 代码实现要点
1. 正确处理空数组
2. 维护堆中元素的索引信息
3. 注意边界条件
4. 高效合并两个有序数组

## 优化可能

2. 并行处理多个子问题
3. 预先分配足够的结果空间
4. 使用迭代器减少内存使用
